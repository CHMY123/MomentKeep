# MomentKeep 前端开发文档

## 项目概述
MomentKeep（朝暮记）是一个基于Vue3 + uniapp开发的跨平台应用，支持网页端、小程序等多端协同。项目使用TiDB Cloud作为数据库，缤纷云存储用户头像等资源。

## 技术栈
- **前端框架**: Vue 3 (Composition API)
- **跨平台框架**: uniapp
- **状态管理**: Pinia
- **网络请求**: uni.request
- **图表库**: ECharts (用于数据可视化)
- **AI集成**: DeepSeek API

## 核心功能模块

### 1. 登录页面 (Login)
**功能描述**: 用户登录认证，支持用户名密码登录

**页面位置**: `pages/login/login.vue`

**核心功能**:
- 用户名密码输入验证
- 登录认证
- 错误提示

### 2. 注册页面 (Register)
**功能描述**: 用户注册，创建新账户

**页面位置**: `pages/register/register.vue`

**核心功能**:
- 用户信息输入
- 密码确认
- 注册验证

### 3. 首页 (Index)
**功能描述**: 应用主页面，展示侧边栏和主内容区

**页面位置**: `pages/index/index.vue`

**核心功能**:
- Layout布局容器
- 页面导航

### 4. 个人中心 (Profile)
**功能描述**:
- 用户资料显示和管理
- 用户名（唯一标识）、昵称、邮箱、手机号
- 用户注销账户功能
- 修改密码功能

**页面位置**: `pages/profile/profile.vue`

**核心功能**:
- 田字形布局显示个人资料
- 头像上传（缤纷云存储）和悬停效果
- 资料编辑功能（支持修改昵称、邮箱、手机号）
- 账户注销确认流程
- 修改密码功能

### 5. 每日打卡 (Daily Check-in)
**功能描述**:
- 四种打卡类型：早起、睡眠、用餐、运动
- 自动获取当前时间并记录
- 用餐打卡支持：早餐、午餐、晚餐、宵夜及自定义
- 运动打卡支持：系统类型选择及自定义输入
- 时间分布图表展示（垂直柱状图）

**页面位置**: `pages/daily-checkin/daily-checkin.vue`

**打卡类型**:
- 早起打卡（type: early）
- 睡眠打卡（type: sleep）
- 用餐打卡（type: meal）：支持早餐、午餐、晚餐、宵夜、自定义
- 运动打卡（type: exercise）：支持跑步、健身、瑜伽、游泳、自定义

**核心功能**:
- 打卡状态实时显示
- 打卡数据统计（本周、本月）
- 时间分布可视化（按小时区间统计）
- 用餐/运动子类型筛选
- 自定义类型添加

### 6. 今日待办 (Todo)
**功能描述**:
- 待办事项管理
- 添加、编辑、删除、标记完成

**页面位置**: `pages/todo/todo.vue`

**核心功能**:
- 今日待办列表
- 待办添加编辑
- 完成状态切换
- 完成说明补充

### 7. 未来倒计时 (Countdown)
**功能描述**:
- 重要事件倒计时
- 支持自定义目标时间和颜色

**页面位置**: `pages/countdown/countdown.vue`

**核心功能**:
- 倒计时列表展示
- 新增倒计时
- 删除倒计时
- 自动计算剩余时间

### 8. 设置 (Settings)
**功能描述**:
- 个性化应用设置
- 主题模式切换（浅色、深色、柔和）
- 字体大小切换（标准、偏大、偏小）
- 背景设置
- 官方主题背景切换
- 意见反馈提交
- 版本信息展示

**页面位置**: `pages/settings/settings.vue`

**核心功能**:
- 主题切换
- 字体大小调整
- 背景图片上传/选择
- 反馈提交功能

### 9. 关于我们 (About)
**功能描述**: 应用相关信息展示

**页面位置**: `pages/about/about.vue`

## 状态管理 (Pinia Store)

### user.js - 用户状态
**功能**: 管理用户登录状态和用户信息
- 用户登录状态
- Token管理
- 用户信息（用户名、昵称、头像等）
- 登录/登出方法

### app.js - 应用状态
**功能**: 管理应用全局设置
- 主题模式
- 字体大小
- 背景设置

## 工具模块

### cache.js - 缓存工具
**功能**: 数据缓存管理
- fetchWithCache: 带缓存的请求方法
- getCache: 获取缓存
- setCache: 设置缓存
- removeCache: 删除缓存

## API请求说明
项目使用uni.request进行网络请求，认证通过JWT Token实现。

请求头格式:
```
Authorization: Bearer <token>
```

## 组件说明

### Layout.vue
布局组件，包含侧边栏和主内容区域
- 响应式设计，移动端/PC端自适应
- 侧边栏导航
- 用户信息展示

## 注意事项
1. 所有页面都需要在pages.json中配置
2. 用户状态通过Pinia管理，刷新后需重新登录
3. API请求需要携带JWT Token
4. 移动端适配需要考虑各种屏幕尺寸
