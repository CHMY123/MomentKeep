# MomentKeep 数据库设计文档

## 1. 数据库概述

MomentKeep项目使用TiDB Cloud作为数据库，基于MySQL兼容的架构。本设计文档详细说明数据库表结构、字段定义、索引设计以及管理员扩展功能。

## 2. 数据库连接配置

根据`application.yml`配置文件，数据库连接信息如下：

```yaml
spring:
  datasource:
    url: jdbc:mysql://gateway01.ap-southeast-1.prod.alicloud.tidbcloud.com:4000/momentkeep
    username: 3M8M5Vr1pf71qDH.root
    password: KKrZhh0RGEMUq7k6
    driver-class-name: com.mysql.cj.jdbc.Driver
    hikari:
      max-lifetime: 30000
      validation-timeout: 3000
      idle-timeout: 60000
      connection-test-query: SELECT 1
```

## 3. 表结构设计

### 3.1 用户表 (`user`)

| 字段名 | 数据类型 | 约束 | 描述 |
| :--- | :--- | :--- | :--- |
| `id` | `BIGINT` | `PRIMARY KEY, AUTO_INCREMENT` | 用户ID |
| `username` | `VARCHAR(50)` | `UNIQUE, NOT NULL` | 用户名（唯一标识） |
| `nickname` | `VARCHAR(50)` | `NOT NULL` | 昵称 |
| `email` | `VARCHAR(100)` | `UNIQUE, NOT NULL` | 邮箱 |
| `phone` | `VARCHAR(20)` | `UNIQUE` | 手机号 |
| `avatar` | `VARCHAR(255)` | | 头像URL（存储在缤纷云） |
| `password` | `VARCHAR(100)` | `NOT NULL` | 密码（BCrypt加密） |
| `status` | `TINYINT` | `DEFAULT 1` | 状态（1-正常，0-禁用，-1-注销） |
| `role` | `VARCHAR(20)` | `DEFAULT 'USER'` | 角色（USER/ADMIN） |
| `create_time` | `DATETIME` | `NOT NULL, DEFAULT CURRENT_TIMESTAMP` | 创建时间 |
| `update_time` | `DATETIME` | `NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 更新时间 |
| `version` | `INT` | `DEFAULT 0` | 乐观锁版本号 |

## 5. 数据迁移与初始化

### 5.1 表结构初始化

根据`application.yml`配置，表结构初始化通过SQL文件完成：

```yaml
spring:
  sql:
    init:
      mode: never
      schema-locations: classpath:sql/tidb_schema.sql
      data-locations: classpath:sql/data.sql
```

### 5.2 初始化SQL文件 (`tidb_schema.sql`)

```sql
-- 创建数据库（如果不存在）
CREATE DATABASE IF NOT EXISTS momentkeep CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE momentkeep;

-- 用户表
CREATE TABLE IF NOT EXISTS `user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `username` VARCHAR(50) NOT NULL,
  `nickname` VARCHAR(50) NOT NULL,
  `email` VARCHAR(100) NOT NULL,
  `phone` VARCHAR(20),
  `avatar` VARCHAR(255),
  `password` VARCHAR(100) NOT NULL,
  `status` TINYINT DEFAULT 1,
  `role` VARCHAR(20) DEFAULT 'USER',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `version` INT DEFAULT 0,
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_username` (`username`),
  UNIQUE KEY `idx_email` (`email`),
  UNIQUE KEY `idx_phone` (`phone`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 打卡表
CREATE TABLE IF NOT EXISTS `checkin` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `type` VARCHAR(20) NOT NULL,
  `checkin_time` DATETIME NOT NULL,
  `meal_type` VARCHAR(20),
  `exercise_type` VARCHAR(20),
  `custom_type` VARCHAR(50),
  `note` VARCHAR(255),
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `version` INT DEFAULT 0,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_checkin_time` (`checkin_time`),
  KEY `idx_type` (`type`),
  KEY `idx_user_time` (`user_id`, `checkin_time`),
  CONSTRAINT `fk_checkin_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 待办表
CREATE TABLE IF NOT EXISTS `todo` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `title` VARCHAR(100) NOT NULL,
  `description` VARCHAR(500),
  `priority` TINYINT DEFAULT 0,
  `completed` BOOLEAN DEFAULT FALSE,
  `completed_time` DATETIME,
  `completion_note` VARCHAR(500),
  `todo_date` DATE NOT NULL,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `version` INT DEFAULT 0,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_todo_date` (`todo_date`),
  KEY `idx_completed` (`completed`),
  KEY `idx_user_date` (`user_id`, `todo_date`),
  CONSTRAINT `fk_todo_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 倒计时表
CREATE TABLE IF NOT EXISTS `countdown` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `title` VARCHAR(100) NOT NULL,
  `description` VARCHAR(500),
  `target_time` DATETIME NOT NULL,
  `color` VARCHAR(20) DEFAULT '#3498db',
  `sort_order` INT DEFAULT 0,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `version` INT DEFAULT 0,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_target_time` (`target_time`),
  KEY `idx_user_target` (`user_id`, `target_time`),
  CONSTRAINT `fk_countdown_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 反馈表
CREATE TABLE IF NOT EXISTS `feedback` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `content` TEXT NOT NULL,
  `contact` VARCHAR(100),
  `status` TINYINT DEFAULT 0,
  `reply` TEXT,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `version` INT DEFAULT 0,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_status` (`status`),
  KEY `idx_create_time` (`create_time`),
  CONSTRAINT `fk_feedback_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 管理员表
CREATE TABLE IF NOT EXISTS `admin` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `username` VARCHAR(50) NOT NULL,
  `password` VARCHAR(100) NOT NULL,
  `name` VARCHAR(50) NOT NULL,
  `role` VARCHAR(20) DEFAULT 'ADMIN',
  `permissions` VARCHAR(255),
  `status` TINYINT DEFAULT 1,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  `version` INT DEFAULT 0,
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_username` (`username`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 用户设置表
CREATE TABLE IF NOT EXISTS `user_setting` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `theme` VARCHAR(20) DEFAULT 'light',
  `background_image` VARCHAR(255),
  `ai_auto_fill` BOOLEAN DEFAULT TRUE,
  `notifications` BOOLEAN DEFAULT TRUE,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `idx_user_id` (`user_id`),
  CONSTRAINT `fk_setting_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- AI对话记录表
CREATE TABLE IF NOT EXISTS `ai_chat` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `user_id` BIGINT NOT NULL,
  `messages` TEXT NOT NULL,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_user_id` (`user_id`),
  KEY `idx_create_time` (`create_time`),
  CONSTRAINT `fk_ai_chat_user` FOREIGN KEY (`user_id`) REFERENCES `user` (`id`) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 6. 管理员扩展功能

### 6.1 管理员权限体系

| 角色 | 权限 | 描述 |
| :--- | :--- | :--- |
| `SUPER_ADMIN` | 所有权限 | 系统超级管理员，拥有所有操作权限 |
| `ADMIN` | 部分权限 | 普通管理员，拥有用户管理、反馈处理等权限 |

### 6.2 管理员功能模块

1. **用户管理**:
   - 查看用户列表
   - 查看用户详情
   - 禁用/启用用户
   - 重置用户密码
   - 查看用户操作记录

2. **反馈管理**:
   - 查看反馈列表
   - 查看反馈详情
   - 回复反馈
   - 标记反馈状态

3. **系统统计**:
   - 活跃用户统计
   - 打卡数据统计
   - 待办完成统计
   - 系统运行状态

4. **系统设置**:
   - 配置管理
   - 日志管理
   - 备份管理

### 6.3 管理员安全措施

1. **密码安全**:
